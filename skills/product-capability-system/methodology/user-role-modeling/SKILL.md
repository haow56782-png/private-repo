# 用户与角色建模

## 一、适用场景
- 多角色系统设计（C端/B端/后台）
- 权限体系设计
- 用户分层策略
- 产品功能按角色差异化
- 客服/SOP按用户等级分级

## 二、核心模型

### 2.1 用户 vs 角色
```
用户（User）→ 自然人，真实存在的人
角色（Role）→ 权限集合，一个用户可拥有多个角色
身份（Identity）→ 某一场景下的标签，如VIP/普通用户
```

### 2.2 用户分层金字塔
```
        VIP / 高价值
       ╱───────╲      → 专属服务、高杠杆
      ╱ 活跃付费 ╲     → 核心收入、留存重点
     ╱  ＿＿＿＿＿  ╲
    ╱  注册免费   ╲   → 转化漏斗、激活
   ╱  ＿＿＿＿＿＿＿  ╲
  ╱   匿名访客     ╲  → 流量入口
 ╱───────────────────╲
```

### 2.3 用户画像要素
- 人口属性：年龄/地域/职业
- 行为特征：频次/金额/时段
- 动机：赚钱/消遣/学习/社交
- 痛点：手续费高/出金慢/不会用
- 目标：短期盈利/长期投资/打发时间

## 三、设计步骤（SOP）
1. 列出所有与系统交互的角色（前台/后台/第三方）
2. 为每个角色定义权限边界（能做什么/不能做什么）
3. 按用户价值分层（RFM模型或业务规则）
4. 创建 3-5 个典型用户画像
5. 绘制用户旅程图（从获客到留存到流失）
6. 设计角色间交互（客服→用户→管理员）
7. 权限矩阵输出

## 四、标准模板
### 角色权限矩阵
```
| 角色 | 功能A | 功能B | 功能C | 管理后台 |
|------|-------|-------|-------|---------|
| 普通用户 | ✅ | ✅ | ❌ | ❌ |
| VIP用户 | ✅ | ✅ | ✅ | ❌ |
| 客服 | ❌ | ✅ | ✅ | 部分 |
| 管理员 | ✅ | ✅ | ✅ | ✅ |
```

## 五、可复用案例
- 交易平台：散户/VIP/做市商/客服/财务/风控 六角色
- 教育平台：学生/老师/助教/教务/管理员
- 分销系统：供货商/分销商/客户/平台方
