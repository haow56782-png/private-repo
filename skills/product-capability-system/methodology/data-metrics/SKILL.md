# 数据指标体系

## 一、适用场景
- 产品上线后需要定义衡量标准
- 增长/运营需要跟踪关键指标
- 数据看板搭建
- 业务报告/周报/月报指标设计
- 融资/汇报需要核心数据支撑

## 二、核心模型

### 2.1 指标分层（AARRR → KPI）
```
| 层级 | 类别 | 核心指标 |
|------|------|---------|
| 获客 | 流量类 | DAU/MAU、新增用户、获客成本(CAC) |
| 激活 | 转化类 | 注册转化率、首充率、首次交易率 |
| 留存 | 粘性类 | 次日/7日/30日留存、DAU/MAU |
| 收入 | 营收类 | ARPU、ARPPU、LTV、GMV |
| 裂变 | 传播类 | 邀请率、K因子、病毒系数 |
```

### 2.2 交易指标体系
```
交易量: 日/周/月交易额、交易笔数、人均交易频次
流动性: 买卖盘深度、点差、订单簿厚度
风控: 异常交易率、冻结率、申诉率
收入: 手续费收入、净收入、利润率
```

### 2.3 指标设计原则
- 可衡量：有明确的数据来源和计算方式
- 可行动：看到指标变化知道该做什么
- 相关性强：直接反映业务目标
- 及时性：能及时获取，不是 T+30 才看到

## 三、设计步骤（SOP）
1. 明确业务目标（这个阶段最重要的事是什么？）
2. 按层级拆解目标（北极星指标 → 二级指标 → 过程指标）
3. 为每个指标定义计算公式
4. 确定数据来源和采集方式
5. 设定目标值（基于历史/行业/预算）
6. 设计数据看板布局
7. 建立指标预警机制

## 五、可复用案例
- 交易所：核心看板 = 日交易量 + 活跃用户 + 手续费收入 + 提现延迟
- SaaS：MRR + Churn Rate + CAC + LTV/CAC
- 内容平台：DAU + 人均时长 + 留存 + 广告收入
