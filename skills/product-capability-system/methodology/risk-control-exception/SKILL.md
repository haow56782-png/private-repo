# 风控与异常处理

## 一、适用场景
- 金融交易风控
- 反欺诈系统
- 账户安全监控
- 异常交易检测
- 合规审查（KYC/AML）
- 容错与回滚机制

## 二、核心模型

### 2.1 风控层次
```
Layer 1: 账户层    登录异常/异地登录/设备指纹
Layer 2: 行为层    频繁操作/异常频次/机器行为
Layer 3: 交易层    大额交易/异常价格/对敲/刷量
Layer 4: 资金层    洗钱/混币/异常流向/闪电贷攻击
Layer 5: 合规层    KYC/制裁名单/政治人物/地域限制
```

### 2.2 规则引擎结构
```
输入事件 → 规则匹配（规则集） → 风险评分 → 决策动作
                                        ↓
                                   通过/人工审核/拒绝/冻结
```

### 2.3 异常处理策略
| 异常类型 | 处理方式 |
|---|---|
| 订单异常 | 撤销/回滚/补偿 |
| 支付超时 | 自动重试/通知用户 |
| 资金异常 | 冻结账户/人工审核 |
| 系统错误 | 降级/熔断/限流 |
| 数据不一致 | 对账脚本/人工修正 |

## 三、设计步骤（SOP）
1. 识别风险场景（列出所有可能出问题的地方）
2. 定义风险等级（低/中/高/严重）
3. 设计规则引擎（规则 + 权重 + 阈值）
4. 设计处置动作（警告/降级/拒绝/冻结/上报）
5. 设计人工审核流程
6. 设计自动化对账机制
7. 设计熔断与降级策略

## 四、标准模板
### 风控规则表
```
| 规则ID | 名称 | 条件 | 风险分 | 动作 | 优先级 |
|--------|------|------|--------|------|--------|
| RC001 | 异地登录 | IP变化>500km + <30min | 60 | 二次验证 | P0 |
| RC002 | 批量注册 | 同IP>5用户/小时 | 80 | 拒绝 | P0 |
| RC003 | 大额提现 | 单笔>$10,000 | 40 | 人工审核 | P1 |
```

## 五、可复用案例
- 交易所风控：持仓限制/杠杆上限/爆仓自动平仓/异常交易暂停
- 支付风控：同卡频繁交易/金额异常/设备切换
- 客服风控：恶意投诉检测/重复注册/滥用退款
