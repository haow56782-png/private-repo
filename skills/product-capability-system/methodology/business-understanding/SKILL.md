# 业务理解方法

## 一、适用场景
- 进入新行业/新赛道需要快速建立认知
- 新项目启动前的业务调研
- 竞品分析与差异化定位
- 商业模式设计/验证
- 与客户/ stakeholder 对齐业务目标

## 二、核心模型

### 2.1 业务理解四层结构
```
┌─────────────────────┐
│ 商业模式层          │ → 怎么赚钱、价值链条
├─────────────────────┤
│ 业务逻辑层          │ → 核心流程、角色、规则
├─────────────────────┤
│ 系统交互层          │ → 用户操作、系统响应
├─────────────────────┤
│ 数据层              │ → 实体、状态、字段
└─────────────────────┘
```

### 2.2 商业模式画布（简化版）
| 模块 | 问题 |
|---|---|
| 价值主张 | 解决什么问题？ |
| 目标用户 | 谁付钱？谁使用？ |
| 收入来源 | 怎么收费？ |
| 成本结构 | 主要成本是什么？ |
| 关键资源 | 技术/牌照/供应链？ |
| 关键伙伴 | 上下游依赖谁？ |
| 渠道通路 | 怎么触达用户？ |
| 客户关系 | 留存/复购/客服？ |

### 2.3 利益相关者矩阵
```
              影响力: 高             低
         ┌──────────────┬──────────────┐
   兴趣:高│ 关键参与者    │ 需要关注者   │
         │ (重点沟通)    │ (定期同步)   │
         ├──────────────┼──────────────┤
   兴趣:低│ 保持满意者    │ 最小关注者   │
         │ (结果同步)    │ (常规通知)   │
         └──────────────┴──────────────┘
```

## 三、设计步骤（SOP）
1. 明确调研目标：验证假设 / 发现机会 / 理解约束
2. 行业扫描：市场规模、增速、竞争格局、监管环境
3. 利益相关者识别：列出所有角色，分析动机
4. 绘制商业模式画布：一页纸讲清怎么赚钱
5. 识别核心业务规则：合规要求、资金限制、风控红线
6. 输出业务理解文档：结构化记录发现
7. 与团队对齐：RAT会议确认理解一致

## 四、标准模板
### 业务调研输出模板
```
# 业务理解报告
## 行业概况
## 目标用户
## 商业模式
## 核心流程
## 竞品分析
## 风险与约束
## 关键发现
## 行动建议
```

## 五、可复用案例
- 进入加密货币交易所行业：快速建立交易/清算/风控认知
- 二元期权平台：理解交易标的、盈利模式、监管红线
- AI Agent 平台：区分 C端 / B端用户，订阅制 vs 按量付费
- 游戏内经济系统：虚拟货币发行、消耗、回收闭环
