# 权限与后台系统

## 一、适用场景
- 多角色后台系统设计
- 运营后台 / 风控后台 / 财务后台
- 权限分级管理
- 操作审计与追溯
- 审核工作流

## 二、核心模型

### 2.1 RBAC 权限模型
```
用户 ←→ 角色 ←→ 权限 ←→ 资源
            ↓
        角色继承：管理员 > 运营 > 客服
        数据权限：全部/部门/仅自己
```

### 2.2 后台标准模块
```
┌─────────────────────────────────────┐
│  仪表盘（Dashboard）                  │
│  ├─ KPI 总览                         │
│  └─ 实时监控                          │
├─────────────────────────────────────┤
│  用户管理                             │
│  ├─ 用户列表/搜索                     │
│  ├─ 用户详情/操作日志                  │
│  └─ 用户分级/标签                     │
├─────────────────────────────────────┤
│  交易管理                             │
│  ├─ 订单管理                          │
│  ├─ 流水查询                          │
│  └─ 异常订单处理                       │
├─────────────────────────────────────┤
│  风控管理                             │
│  ├─ 规则配置                          │
│  ├─ 风控事件                          │
│  └─ 黑白名单                          │
├─────────────────────────────────────┤
│  系统配置                             │
│  ├─ 参数管理                          │
│  ├─ 权限配置                          │
│  └─ 操作日志                          │
└─────────────────────────────────────┘
```

### 2.3 审核流程（审批流）
```
提交 → 初审 → 复审 → 终审 → 生效
  ↓      ↓      ↓      ↓
驳回   驳回   驳回   驳回
```

## 三、设计步骤（SOP）
1. 枚举后台角色及其职责边界
2. 定义资源树（菜单/按钮/数据）
3. 配置每个角色的权限矩阵
4. 设计审核流程（单审/双审/多级）
5. 设计操作日志记录规范
6. 设计数据权限隔离（跨部门数据隔离）

## 四、标准模板
### 操作日志字段
```
operator_id | action | resource_type | resource_id | detail | ip | user_agent | result | created_at
```

### 审核记录表
```
audit_id | order_id | reviewer | action(approve/reject) | comment | created_at
```

## 五、可复用案例
- 交易所后台：用户冻结/解冻、交易暂停、参数配置、审核提现
- 客服后台：工单流转、用户信息查询、操作记录审计
- 分销后台：佣金结算审核、代理商等级调整
