# AI Agent 产品范式

## 一、适用场景
- AI Agent / 智能助手产品设计
- Agent 技能系统架构
- 多 Agent 协作流程
- AI 客服 / Copilot / 自动化工作流
- 大模型落地的产品化设计

## 二、核心模型

### 2.1 Agent 循环（感知-推理-行动）
```
用户输入
   ↓
感知（Perception）→ 理解意图、提取实体
   ↓
推理（Reasoning）→ 拆解任务、选择策略
   ↓
行动（Action）   → 调用工具、生成回复
   ↓
反馈（Feedback） → 观察结果、判断是否完成
   ↓
循环直到任务完成
```

### 2.2 工具调用设计
```
| 工具类型 | 示例 | 设计要点 |
|---------|------|---------|
| 读取类 | 查数据/查文档 | 明确的入参和返回值 |
| 写入类 | 创建订单/发送消息 | 确认机制防误操作 |
| 计算类 | 分析/统计/预测 | 必须有执行边界 |
| 外部类 | 调API/查网页 | 超时和降级处理 |
```

### 2.3 Human-in-the-Loop 模式
```
| 等级 | 名称 | 自动程度 | 适用场景 |
|------|------|---------|---------|
| L1 | 全自动 | 不需要人类 | 查询/推荐/通知 |
| L2 | 建议+确认 | Agent生成→人类确认→执行 | 交易/发消息/写内容 |
| L3 | Agent起草→人工修改 | Agent出草稿→人改→人发 | 客服回复/报表/合同 |
| L4 | 人工操作 | Agent仅提供信息 | 高风险/首次操作 |
```

## 三、设计步骤（SOP）
1. 定义 Agent 的职责边界（能做什么/不能做什么）
2. 设计感知层（用户输入怎么解析）
3. 设计推理策略（单步执行 vs 多步规划）
4. 设计工具集（每个工具的 SKILL.md）
5. 设计反馈机制（怎么知道做完了？）
6. 设计异常处理（超时/误解/拒绝/升级给人）
7. 设计记忆策略（短期上下文/长期记忆/知识库）

## 五、可复用案例
- 客服 Agent：接收问题 → 查知识库 → 生成回复 → 人工确认 → 发出
- 交易分析 Agent：用户问行情 → 查链上数据 → 分析趋势 → 生成报告
- 数据查询 Agent：用户问"上周成交量" → 查数据库 → 计算 → 回答问题
