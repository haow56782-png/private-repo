# 增长与分销机制

## 一、适用场景
- 用户增长策略设计
- 分销/代理体系搭建
- 裂变活动设计
- 积分/等级/权益体系
- 用户召回策略

## 二、核心模型

### 2.1 AARRR 增长漏斗
```
Acquisition（获客）   → 广告/ASO/KOL/裂变
Activation（激活）    → 注册/首充/首次交易
Retention（留存）     → 签到/复购/消息触达
Revenue（收入）       → 付费/升级/手续费
Referral（裂变）      → 邀请/分销/返佣
```

### 2.2 分销层级模型
```
一级：A 邀请 B → A 获直接佣金
二级：B 邀请 C → A 获间接佣金，B 获直接佣金
三级：...（多数合规限制在两级以内）
```

### 2.3 激励机制设计
| 类型 | 适用场景 | 策略 |
|---|---|---|
| 现金返佣 | 分销/代理 | 按交易额比例 |
| 积分/点券 | 用户活跃 | 签到/任务兑换 |
| 等级权益 | VIP体系 | 升级解锁福利 |
| 锁仓/Staking | 平台代币 | 持有分配手续费 |

## 三、设计步骤（SOP）
1. 确定增长目标（新增/留存/收入哪个优先）
2. 选择获客渠道（自然流量/付费/裂变/分销）
3. 设计激活路径（从注册到首次核心操作的路径）
4. 设计留存机制（推送/签到/订阅/权益）
5. 设计裂变机制（邀请码/返佣/团队奖）
6. 设定成本模型（CAC/LTV 算账）
7. 配置激励结算规则

## 四、标准模板
### 分销佣金计算表
```
| 层级 | 比例 | 条件 |
|------|------|------|
| 一级 | 20% | 直接邀请用户手续费 |
| 二级 | 5% | 下级邀请的用户手续费 |
| 团队 | 2% | 团队总交易额超额部分 |
```

## 五、可复用案例
- 加密货币交易所邀请返佣：20%手续费返佣+下级返佣
- 电商分销：一级10%、二级3%
- 游戏推广：下载→注册→充值三级奖励
- SaaS 推荐：推荐成功赠送 1 个月免费
