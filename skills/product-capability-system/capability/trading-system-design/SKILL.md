# 交易系统设计能力

## 一、适用场景
- 现货/合约交易所
- 外汇/差价合约平台
- 二元期权交易系统
- 做市商系统
- 订单路由系统

## 二、核心模型

### 2.1 交易系统架构
```
用户层: App / Web / API
    ↓
接入层: 认证 / 限频 / 签名验证
    ↓
交易引擎层:
  ┌─────────────────────────┐
  │ 订单管理 → 风控检查      │
  │    → 订单簿匹配          │
  │    → 成交处理 → 清算入账  │
  └─────────────────────────┘
    ↓
基础设施层: 账户 / 账本 / K线 / 行情
```

### 2.2 订单类型
| 类型 | 说明 | 匹配策略 |
|------|------|---------|
| 市价单 | 立即成交 | 最优价格直到成交 |
| 限价单 | 指定价格 | 价格匹配 + 时间优先 |
| 止损单 | 触发后转市价 | 市场价触发 |
| OCO | 二选一 | 任一成交取消另一 |

### 2.3 订单簿设计
```
卖盘 (Asks)
价格 ↑ 量
100.5  1.0
100.4  2.5
-----------------
买盘 (Bids)
100.2  3.0
100.1  1.5
价格 ↓ 量
```

## 三、设计步骤（SOP）
1. 定义交易品种（现货/合约/二元期权？）
2. 设计订单类型和状态机
3. 设计订单簿（Price-Time Priority）
4. 设计风控检查点（下单前/成交前/成交后）
5. 设计清算入账流程
6. 设计行情推送（Tick / K线 / 实时深度）

## 四、标准模板
### 订单状态机
```
Created → PendingRiskCheck
         → Open（挂单中）
         → PartiallyFilled（部分成交）
         → Filled（全部成交）
         → Cancelled（已撤单）
         → Expired（已过期）
```

## 五、可复用案例
- 币币交易所：BTC/USDT 现货交易，限价+市价
- 合约交易所：BTC永续合约，全仓/逐仓保证金
- 二元期权：60秒/5分钟/1小时到期产品
