# 风控系统设计能力

## 一、适用场景
- 交易平台整体风控架构
- 实时交易监控系统
- 反欺诈/反洗钱平台
- 合规管理系统
- 多层级风控策略

## 二、核心模型

### 2.1 风控系统架构
```
数据接入层: 交易/登录/提现/行为事件
    ↓
规则引擎层: 规则配置 → 规则编译 → 规则执行
    ↓
风险评分层: 单规则分 → 累加 → 综合评分
    ↓
决策执行层: 通过/拒绝/审核/冻结/降级
    ↓
案件管理层: 人工审核 → 处置记录 → 复盘优化
```

### 2.2 风控策略矩阵
```
| 用户等级 | 小额交易 | 大额交易 | 提现 | 登录 |
|---------|---------|---------|------|------|
| 新用户 | 通过 | 人工审核 | 24h冷却 | 二次验证 |
| 普通用户 | 通过 | 通过 | 1h冷却 | 二次验证 |
| VIP | 通过 | 通过 | 通过 | 通过 |
| 高风险 | 审核 | 冻结 | 冻结 | 冻结 |
```

### 2.3 额度管理
```
单笔限额: $10,000
日累计限额: $50,000
月累计限额: $500,000
冷却时间: 大额提现后24h内不可再提
```

## 三、设计步骤（SOP）
1. 梳理所有风控场景（交易/登录/提现/行为）
2. 设计规则引擎架构（可配置+热加载）
3. 设计风险评分模型
4. 设计用户分层风控策略
5. 设计人工审核工作台
6. 设计自动处置流程
7. 设计风控数据看板

## 四、标准模板
### 风控事件表
```
event_id | event_type | user_id | risk_score | decision | handler | created_at
```

## 五、可复用案例
- 交易所三层风控：账户层(登录/设备) → 交易层(价格/量/频次) → 资金层(流向/洗钱)
- 电商反欺诈：同地址多账号/货到付款不取/虚假交易
- 支付风控：金额异常/频次异常/设备指纹匹配
