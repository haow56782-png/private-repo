# Agent 任务建模能力

## 一、适用场景
- 把现有业务流程转化为 Agent 自动执行
- 设计多步骤 Agent 任务链
- 工具编排与依赖管理
- 人机协作流程设计
- Agent 技能系统架构

## 二、核心模型

### 2.1 任务分解模型
```
用户需求
   ↓
任务链（Task Chain）→ 有序的任务序列
   ├── Task 1: 理解需求 → 工具: 无（推理）
   ├── Task 2: 查数据   → 工具: 数据库查询
   ├── Task 3: 分析     → 工具: 计算引擎
   └── Task 4: 输出     → 工具: 生成器
   ↓
每个任务: 输入 → 工具 → 输出 → 判断
```

### 2.2 工具编排模式
```
| 模式 | 描述 | 示例 |
|------|------|------|
| 串行 | 前一个输出=后一个输入 | 查数据→分析→出报告 |
| 并行 | 多个独立工具同时调用 | 同时查行情+查链上数据 |
| 条件 | 根据结果选择不同路径 | 是VIP→VIP策略/否则→标准策略 |
| 循环 | 重复直到条件满足 | 持续监控价格直到触发 |
```

### 2.3 Human-in-the-Loop 节点
```
流程中插入「人工确认」节点：

[自动执行] → [人工确认] → [继续自动]
              ↑       ↓
             拒绝    通过/修改
```

## 三、设计步骤（SOP）
1. 把业务需求拆成原子步骤（不能被继续拆分的最小单元）
2. 标注每个步骤的「工具依赖」（需要什么才能执行）
3. 标注每个步骤的「人工介入点」（哪里需要人确认）
4. 设计异常处理（某一步失败怎么办）
5. 设计上下文传递（前一步输出怎么给下一步）
6. 测试完整任务链

## 四、标准模板
### 任务链定义
```
Task Chain: [名称]
Trigger: [触发条件]

Tasks:
1. [任务名] → Tool: [工具] → Output: [产出]
2. [任务名] → HITL: [人工确认点]
3. [任务名] → Tool: [工具] → Output: [产出]

Fallback: [某个步骤失败时的降级方案]
```

## 五、可复用案例
- 客服Agent链：接收问题→查知识库→生成回复→人工确认→发出
- 交易分析链：用户输入口号→查K线→查链上→分析→出报告
- 报表生成链：设定参数→拉数据→计算→格式化→发送
