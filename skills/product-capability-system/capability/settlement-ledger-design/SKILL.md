# 结算与账本设计能力

## 一、适用场景
- 多币种/多资产结算系统
- 分账系统（多方分润）
- 自动化对账系统
- 财务审计系统
- 差错处理

## 二、核心模型

### 2.1 结算体系架构
```
交易产生
   ↓
T+0模块: 实时入账（可用余额更新）
T+1模块: 批量清算（日切+对账）
T+N模块: 月结/季结/年结
   ↓
输出: 结算报表 / 对账结果 / 差错单
```

### 2.2 账本核心规则
```
| 科目 | 借方 | 贷方 | 说明 |
|------|------|------|------|
| 用户A账户 | 100 | | 用户A充值 |
| 平台收入 | | 2 | 手续费 |
| 用户B账户 | | 98 | 结算给B |
```

### 2.3 对账机制
```
三方对账:
平台账本 ⇔ 支付渠道 ⇔ 银行/区块链
        ↓
差异处理: 挂账 → 人工确认 → 调账
```

## 三、设计步骤（SOP）
1. 定义结算对象（用户/商家/代理/平台）
2. 定义结算周期（实时/T+0/T+1/月结）
3. 设计账本结构（多币种+科目体系）
4. 设计分账规则（固定比例/阶梯/按条件）
5. 设计对账流程（渠道对账+内部对账+差错处理）
6. 设计结算报表

## 四、标准模板
### 结算试算表
```
| 项目 | 金额 | 说明 |
|------|------|------|
| 期初余额 | 100,000 | |
| + 收入 | 50,000 | 今日手续费 |
| + 入金 | 200,000 | 用户充值 |
| - 出金 | 30,000 | 用户提现 |
| - 手续费 | 2,000 | 渠道费用 |
| = 期末余额 | 318,000 | |
```

## 五、可复用案例
- 交易所结算：逐笔实时清算+日终对账+月度财务报表
- 分销结算：下级出金→上级佣金实时入账→月度结算单
- 分账系统：接入第三方支付→自动按比例分账给多方
